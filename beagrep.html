1 Introduction

I still remember when I read the ``Linux Device Drivers'', 1st Edition, in the preface, the author says:

… The text you are approaching is the result of hours of patient grepping …

I have been using grep to read source code ever since. It's been adequate for a long time, until I started reading Android source code with it. I remember vividly how it took me 30 minutes to grep readlink in Android source code (with VCS directory and binary files excluded).

Then I learned about the beagle software, and it occured to me, what if I use beagle and grep together? I mean, to use beagle first to decide the (relatively a lot smaller) set of possible matching files (i.e., files that contained the word readlink, taking the previous example), then invoke grep on this set only. Usually the job can be done in a couple of seconds.

Beagrep supports both Linux and Windows (requires CYGWIN).

2.1.2 Download Code

git clone git://github.com/baohaojun/beagrep.git -b Linux

2.1.3 Compile and Install

In the beagrep directory you just git cloned: 

cd beagrep
sh ./autogen.sh
make
sudo make install

2.2 Windows Install

2.2.1 Dependencies

  • Install cygwin.
  • Install mono.

2.2.2 Download Code

git clone git://github.com/baohaojun/beagrep.git -b Windows

2.2.3 Compile and Instal

cd beagrep
bash build-win.sh

3 Usage

3.1 Create Index

In the source code directory you want to ``beagrep'', for e.g. android, 

cd ~/src/android
mkbeagrepidx

This will take quite some time (about the same amount of time you take to run grep directly in the source code directory, i.e., ~0.5 hour).

3.2 run beagrep

In the source code directory (or any subdirs under it) 

cd ~/src/android
beagrep -e "readlink"
